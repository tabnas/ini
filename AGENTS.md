# Agents Guide ‚Äî ini

## What this project is

`@tabnas/ini` is an **INI grammar plugin** for the
[`tabnas`](https://github.com/tabnas/parser) parsing engine. It parses
[INI](https://en.wikipedia.org/wiki/INI_file) text into objects / maps ‚Äî
sections (`[a]`), dotted/nested section paths (`[a.b.c]`), `key = value`
pairs, repeated-key arrays (`tags[] = x`), bare boolean keys, quoted
values, line and inline comments, and multiline values (backslash
continuation or indent continuation).

It is **not** standalone. Like the other tabnas grammar plugins it layers
on the [`@tabnas/jsonic`](https://github.com/tabnas/jsonic) base grammar
(the relaxed-JSON `val`/`map`/`list`/`pair`/`elem` rules) and reuses the
engine's lexer and comment handling. INI values are not JSON, so the
plugin also depends on [`@tabnas/hoover`](https://github.com/tabnas/hoover)
to lex raw value / key / dive-key spans (the `endofline`, `key`, and
`divekey` blocks) instead of jsonic's text/string/number matchers.

There are two implementations that must behave identically ‚Äî TypeScript
(canonical) and a Go port.

There is **no CLI** in this repo ‚Äî `ts/package.json` has no `bin`, and
`src/` holds only `ini.ts`.

## The tabnas engine dependency

Both runtimes depend on the unpublished `@tabnas` siblings via a
**sibling checkout** (the standard tabnas dev model until the packages
publish tagged releases):

- TypeScript: `@tabnas/parser`, `@tabnas/jsonic`, and `@tabnas/hoover` are
  declared as `peerDependencies` (`parser`/`jsonic` `">=2"`, `hoover`
  `">=0"`) in [`ts/package.json`](ts/package.json) and mirrored as
  `file:../../parser/ts` / `file:../../jsonic/ts` / `file:../../hoover/ts`
  devDependencies for local builds. `@tabnas/debug` and `@tabnas/railroad`
  are dev-only `file:` devDependencies (debug for the `debug.model()`
  test, railroad to regenerate `ts/doc/grammar.{svg,txt}`).
- Go: `go/go.mod` requires `github.com/tabnas/hoover/go` and
  `github.com/tabnas/jsonic/go` directly, with `parser`, `json`, and
  `debug` as indirect requires. All five are wired with `replace
  github.com/tabnas/<dep>/go => ../../<dep>/go`. `ini.go` imports
  `jsonic` (not `parser`) and `hoover` ‚Äî jsonic re-exports the engine
  types in Go.

Clone the siblings (`parser`, `jsonic`, `hoover`, plus `debug`/`railroad`
for the optional tests) next to this repo and build their TS first. CI
does this for you (see below).

## Authority and alignment rules

**TypeScript is canonical. Go is a port of it.** When you change
behaviour:

1. Change `ts/src/ini.ts` first (or `ts/ini-grammar.jsonic` for grammar
   changes ‚Äî see the embed section below).
2. Port the same change to `go/ini.go`.
3. Add/extend the shared fixture(s) in `test/spec/*.tsv` so both runtimes
   assert the new behaviour. The fixtures are the parity contract; both
   suites resolve them by path (TS:
   [`ts/test/ini-tsv.test.ts`](ts/test/ini-tsv.test.ts) reads
   `../../test/spec`; Go: [`go/ini_tsv_test.go`](go/ini_tsv_test.go)
   `tsvSpecDir()` returns `../test/spec`).
4. Mirror any new unit cases across `ts/test/ini.test.ts` and
   `go/ini_test.go` ‚Äî the two unit suites should cover the same ground.
5. Run both suites and confirm green.

## The grammar is embedded ‚Äî never hand-edit the embedded block

`ts/ini-grammar.jsonic` is embedded verbatim into **both** `ts/src/ini.ts`
and `go/ini.go`, between these markers:

```
// --- BEGIN EMBEDDED ini-grammar.jsonic ---
...
// --- END EMBEDDED ini-grammar.jsonic ---
```

Edit `ts/ini-grammar.jsonic`, then run the embed step. Never edit the text
between the markers by hand ‚Äî it will be overwritten. (The grammar may not
contain backticks; `embed-grammar.js` aborts if it does, since the Go side
embeds it as a raw string.)

```bash
cd ts && npm run embed   # node embed-grammar.js ‚Üí writes ts/src/ini.ts AND go/ini.go
```

`npm run build` runs the embed step first, so a normal TS build keeps both
files in sync.

The grammar file defines the `ini`, `table`, `dive`, `map`, and `pair`
rules plus the static `options` block (start rule, excluded jsonic
alternatives, fixed tokens, comment defs, the `@line-check` callback).
The `val` rule is configured **in code instead** (`tn.rule('val', ...)` in
TS / its Go equivalent), because it needs a custom alt-filter and
injection modifier the grammar spec can't express. `@`-prefixed names in
the grammar resolve against the `refs` map (state actions are auto-wired
by the `@rulename-{bo,ao,bc,ac}` convention via `fnref`; alt actions,
conditions, error handlers, and the `@line-check` option callback are
referenced explicitly).

## Pruned jsonic rules

INI has no array syntax, so the `val` rule is restricted to scalars and
maps, leaving jsonic's inherited `list` and `elem` rules **unreachable**.
The plugin deletes them so the parser ‚Äî and its railroad diagram ‚Äî only
contains the rules INI actually uses:

- TS (`src/ini.ts`): `for (const name of ['list', 'elem']) tn.rule(name, null)`.
- Go (`go/ini.go`): `for _, name := range []string{"list", "elem"} { j.Rule(name, nil) }`.

The grammar also nulls jsonic's `#OB` / `#CB` / `#CL` fixed tokens (and
the `#OS`/`#CS` brackets stay only as section-header delimiters). When you
add or remove a rule, update the rule-name set asserted by the
`debug.model()` test (currently `['dive', 'ini', 'map', 'pair', 'table',
'val']`).

## Architecture notes

- **Hoover does the value lexing.** The plugin `tn.use(Hoover, ...)` with
  three blocks: `endofline` (the `#HV` value span, terminated by newline
  and ‚Äî when inline comments are active ‚Äî comment chars), `key` (the `#HK`
  property key, up to `=`), and `divekey` (the `#DK` dotted section-path
  segment, up to `]` or `.`). Comment-char terminators and backslash
  escape maps are built dynamically from the `comment.inline` options.
- **Multiline values** (`multiline` option) and **whitespace-prefixed
  inline comments** install a custom lex matcher (`order: 8.4e6`, just
  below Hoover's `8.5e6`) that intercepts the value, handling backslash
  continuation, indent continuation, and literal/escaped comment chars,
  then emits an `#HV` token tagged `use.block = 'endofline'`.
- **Duplicate sections** (`section.duplicate`: `merge` | `override` |
  `error`) are tracked in a per-parse `declaredSections` set, cleared in
  the `ini` rule's bo handler.
- The embedded grammar text is parsed by a **separate** jsonic engine
  instance (`new Tabnas().use(jsonic).parse(grammarText)`), its `ref` /
  `options.string.chars` are set, then it is handed to `tn.grammar(...)`.
- Token legends: the plugin registers human descriptions for INI tokens
  (`#DK`, `#HK`, `#HV`) via `config.modify` ‚Üí `cfg.tokenDesc`, which
  `@tabnas/railroad` reads off the live config when drawing the diagram
  legend.

## Build & test

TypeScript (from `ts/`):

```bash
npm install            # auto-installs peers; resolves file: siblings
npm run build          # embeds grammar, builds the hoover sibling's tsconfig, then tsc --build src test
npm test               # node --test over dist-test/*.test.js
```

Note `npm run build` runs `tsc -p node_modules/@tabnas/hoover/src/tsconfig.json`
before `tsc --build src test`, so the linked Hoover sibling is compiled as
part of this build.

Go (from `go/`):

```bash
go build ./...
go test -v ./...       # unit tests + shared fixtures
```

The repo root [`Makefile`](Makefile) wraps both: `make build|test` run the
TS and Go halves, and `make publish-go V=x.y.z` injects `V` into the
`const Version` in `go/ini.go` (currently `0.1.6`) and tags `go/vX.Y.Z`.
The TS package version is `0.9.1` ([`ts/package.json`](ts/package.json)).
Local builds resolve the unpublished siblings via the repo-set `go.work` +
`node_modules` symlinks created by `admin/scripts/link.sh` (there is no
checked-in `go.work` here).

## Optional test (@tabnas/debug)

[`ts/test/debug-model.test.ts`](ts/test/debug-model.test.ts) composes the
ini plugin with the external `@tabnas/debug` plugin and asserts the
structured grammar model: the rule set (`['dive', 'ini', 'map', 'pair',
'table', 'val']`), `m.config.start === 'ini'` (note `config.start`, not
`m.start`), that `Ini` is in `m.plugins`, and the push edges (`ini` ‚Üí
`table`; `table` ‚Üí `dive` and `map`; `pair` ‚Üí `val`). It resolves debug
dynamically and **skips** unless `@tabnas/debug` is installed (it is a
`file:` devDependency, so `npm test` runs it) or `TABNAS_DEBUG_PATH`
points at a built sibling.

[`ts/test/doc-examples.test.ts`](ts/test/doc-examples.test.ts) extracts
fenced examples containing `// =>` assertions from the README and docs and
runs them ‚Äî keep doc examples correct.

## CI

[`.github/workflows/build.yml`](.github/workflows/build.yml) has two jobs,
neither publishing to npm:

- **build** (Ubuntu/Windows/macOS, Node 24): sets `git config --global
  core.autocrlf false` (CRLF corrupts the `.tsv` fixtures), git-clones the
  tabnas closure (`parser debug json hoover abnf railroad jsonic`) as
  siblings, `npm i && npm run build --if-present` each in topo order, then
  `npm test` here.
- **build-go** (Ubuntu/macOS, Go 1.24): clones the same siblings, mirrors
  `admin/scripts/link.sh` by creating `vendor/` symlinks for any
  `../vendor/` replaces and a `go work` over every non-vendor-replaced
  module, then `go build ./...` / `go test -v ./...` here.
